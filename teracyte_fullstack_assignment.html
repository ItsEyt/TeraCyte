🧪 TeraCyte Full-Stack Home Assignment

Live Image Dashboard with Inference Overlay (Desktop-only UI)

🧬 About the Assignment

At TeraCyte, we build microscopy and AI systems that stream image data and real-time inference. In this assignment you will implement a Python backend and a desktop-only web frontend that authenticate, poll protected endpoints on the hosted TeraCyte server, and render images, metrics, and a histogram.

🎯 Your Mission

Build a full-stack app consisting of:

1) Backend (Python) — act as a thin proxy/auth client to the hosted API and manage JWT lifecycle:

  • POST /api/auth/login — obtain access & refresh tokens
  • POST /api/auth/refresh — renew access token
  • GET /api/auth/me — get current user
  • GET /api/image — fetch latest microscope image (base64 PNG) + image_id
  • GET /api/results — fetch metrics for that image_id
  • Use the hosted TeraCyte server below.

2) Frontend (any framework) — desktop-only UI that:

  • Authenticates with JWT and auto-refreshes tokens
  • Polls /api/image and /api/results periodically
  • Detects new image_id and updates the view
  • Displays the image, metrics (intensity_average, focus_score, classification_label)
  • Renders a 256-bin histogram (chart)

🛠️ Technical Guidelines

  • Backend: Python (FastAPI / Flask / Django REST). Implement JWT + refresh as a client to the hosted API.
  • Frontend: Any framework (React / Vue / Angular / etc.). Desktop layout only.
  • Local DB: Persist fetched results locally (e.g., SQLite recommended). Use it to power the History panel and Stats. Provide either the .db file, a dump, or a docker-compose.yml to recreate it.
  • Apply one classical image-processing step; provide an Original/Processed toggle.
  • Only update when image_id changes; render a 256-bin histogram.
  • Secure token storage; graceful 401 handling and retry logic.
  • Prefer typed models (pydantic / dataclasses) and clean separation of concerns.
